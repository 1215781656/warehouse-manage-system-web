# 部署分工与协作指南

为了将本项目顺利部署到腾讯云服务器，我们采用 **“本地构建 + 上传产物”** 的模式。这种模式能有效避免服务器因内存不足而构建失败的问题。

以下是我们双方的分工：

## 🤖 我 (AI 助手) 负责的工作

1.  **环境配置**：我已经为你创建了所有必要的服务器配置文件：
    *   `nginx.conf`: 用于配置 Nginx Web 服务器，反向代理前端和后端 API。
    *   `server/Dockerfile`: 用于构建后端的运行环境（轻量级，只安装生产依赖）。
    *   `docker-compose.yml`: 一键启动所有服务（前端 Nginx + 后端 Node + 数据库 MySQL）。
2.  **操作指南**：编写详细的《部署操作手册》，指导你每一步如何操作。
3.  **故障排查**：如果你在部署过程中遇到报错，我会根据日志协助你解决。

## 👨‍💻 你 (用户) 需要做的工作

1.  **本地构建**：
    *   在本地电脑上运行命令，编译前端和后端代码，生成 `dist` 目录。
2.  **文件上传**：
    *   使用文件传输工具（如 FileZilla, WinSCP 或 SCP 命令）将指定的文件上传到腾讯云服务器。
3.  **服务器操作**：
    *   登录腾讯云服务器（SSH）。
    *   安装 Docker（如果还没安装）。
    *   运行一条命令启动项目。
